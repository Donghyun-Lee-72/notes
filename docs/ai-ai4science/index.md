# AI & AI4Science

This section connects core AI concepts to systems that retrieve evidence, use
tools, run workflows, and support scientific work.

## Foundations

Start with language models, tokenization, transformers, context windows,
embeddings, and prompting. These pages establish the representation and
inference layer used by later systems.

## Engineering systems

The build notes cover retrieval-augmented generation, reranking, structured
outputs, tool use, agent memory, workflows, and AI agents. They emphasize that
application code—not the model—owns authorization, validation, execution, and
stopping conditions.

## AI4Science

AI4Science applies these capabilities to scientific discovery, simulation,
experiment planning, literature synthesis, and data analysis. Domain validity
requires more than fluent output: systems need traceable evidence, units and
conditions, uncertainty, reproducible tools, and evaluation against scientific
ground truth. Future pages will connect these principles to materials science,
chemistry, and scientific instruments.

## Evaluation

Use the grounding, LLM evaluation, and RAG evaluation pages to separate source
quality, retrieval, reasoning, generation, and tool-execution failures.
